1. System Requirements

Real simple, you need to have the Java Runtime Environment version 1.3.0 or later installed. This is packaged Netscape 6+ and some versions of Linux.

Currently, the Java Runtime Environment is available at http://java.sun.com/j2se/1.3/jre/.

Presently, Hapsby has only been tested under Windows 2000. However, this program should work under any operating system with Java compatibility.

2. Installation

Extract the file hapsby.zip, make sure you extract with the full path ("Extract Files with Full Path" or "Use Folder Names" are typical options). From here you can either run the executable jar file, or build the source yourself.

Running under Windows 95/98/2000/NT: navigate to the \bin\win32 folder and run the file Hapsby.bat. Right now, Windows 2000 seems to be able to run a jar file by just double-clicking it; if that doesn't work for you, use the .bat file.

Running under other platforms: Please check the Java documentation for running a jar executable under your operating system. The Hapsby.jar file is located in the \bin folder. Typically you would enter "java -jar Hapsby.jar" from a command line.

Building the source: You must have JDK1.3.0 (or later) installed. See the file \source\build-win32.bat for an example of how to properly build the source. 

3. Using Hapsby

This is a very brief quick start guide.... please see the help folder for a real guide.

Select a save game to browse for from the list on the left. Pressing the "Open Save Game" button or menu will open the save game browser. Navigate to the file you want to edit and open it.

The game description and properties will load in the right panel. Select a property to edit and the panel will update with the information about that property. All that's left is to enter a new value in the "Current Value" box, and press the "Save" button. 

4. Creating save game definitions (.hsd)

This is a very basic guide, see the help file for a better tutorial.

When saving the files do not use blank spaces in the filename. Not all Java compatible operating systems support spaces in file names, therefore Hapsby does not support spaces in file names.

Hapsby save definition files must be in this form:

Game Title

Game Description

Save File Name/Pattern

Total Number of Properties

Property1 Name

Property1 Description

Property1 Data Type

Property1 Address (in decimal representation)

Property1 Length (in decimal representation)

Property1 Minimum Value (in decimal representation)

Property1 Maximum Value (in decimal representation)

Description of Fields:

Game Title: Title of the game.

Game Description: Description of game. Include information about what type of save state this is, and other relevant information. May be in html format.

Save File Pattern: What the name/pattern of the save game is. For example: Nesticle save states would be *.st*, and Genecyst save states would be *.gs*. Wildcards can only be leading or trailing - something like *.g*0 will not work.

Total Number of Properties: Number of properties. Incorrect property count value will cause problems. 

Property (n) Name: Name of the property.

Property (n) Description: Description of property. May be in html format.

Property (n) Data Type: Presently, the only valid data types value are 1 (integer, low byte stored first), and 2 (integer, high byte stored first). Future versions will hopefully support character/string data types.

Property (n) Address: Starting address of property. Must be in decimal format. For example, and address starting at 0BC1 would have an address of 3009.

Property (n) Length: Number of bytes. Just about every NES or SMS game uses 1 byte (8-bit) for everything (except for experience points in RPGs). Just about every Genesis game uses 2 bytes (16-bit) for everything.

Property (n) Minimum Value: Minimum value for the property, usually 0.

Property (n) Maximum Value: Maximum value for the property. Usually it's something like 255 or 65535, it's a good idea to check the manual in advance to see.

Skipping a line will cause problems.

5. Contacting the author

Here are the guidelines for contacting the me:

  1. Please see if your inquiry is covered in the FAQ - check HuguesJohnson.com for the most recent version

  2. Do not ask any programming questions.

  3. Do not send any files.

  4. Do not ask me to send you any files.

  5. Please include the word "Hapsby" somewhere in your subject line or my junk mail filter will certainly delete your message before I ever see it.

  6. On the topic of junk mail filters, messages with vulgar content will also be automatically deleted.

Any mail sent outside of these guidelines will be deleted.
